# Phase 20 — Adaptive self-learning profile
Status: done
Goal: The app measurably learns how the user learns and reviews — deterministic signals plus one cheap end-of-session reflection maintain human-readable Markdown profile files that adapt both Learning mode and the repo-review side, at near-zero marginal API cost.
Depends on: Phase 17 (provider_for(), FSRS integrity, transactional saves) and Phase 16 (events/cancel infra).

Design provenance (researched 2026-06-09): Honcho, mem0, Letta/MemGPT, LangMem, Zep/Graphiti, and cognee were all evaluated and rejected as dependencies — every one is a multi-process server (and Honcho is AGPL), none outputs the git-diffable Markdown Sam wants, and their per-interaction LLM cost model is wrong for a desktop app. Instead: own reflection loop + MD files, borrowing Honcho's evidence-grounded "dialectic" framing, mem0's single-pass ADD/UPDATE/DELETE reconcile, and Letta's char-capped labeled memory blocks. Sam's personal ~/honcho-local stays personal infra — the app must never depend on it (an optional one-way export of the profile into Honcho is future work).

## Tasks
- [x] **T1 Deterministic signal capture** — add `profile_events` (and aggregation queries) capturing, at existing touchpoints only: quiz accuracy by question `kind`, FSRS stability/lapse aggregates by skill and card format, explanation re-ask counts, session length, suggestion approved/dismissed rates by kind, grill answer length/latency, assess re-runs. No model calls anywhere in this task. Done when: a seeded session produces correct aggregates in SQL, verified by unit tests, with zero ModelProvider invocations.
- [x] **T2 Profile store + MD writer** — `profile/` in app-data with `learner-profile.md`, `review-preferences.md`, `subjects/<slug>.md`; each file has sentinel-marked sections: Facts (regenerated from SQL each write), Observations (model-owned, ≤2,000 chars), Your notes (never auto-edited). Writer rewrites only between markers and refuses malformed input. Done when: round-trip tests prove user-section bytes survive 100 rewrites, caps are enforced, and a hand-edited file is never clobbered.
- [x] **T3 End-of-session reflection call** — one `ModelRequest` (model alias `haiku`, no tools) at session end, gated on ≥15 new events since last reflection; input = deterministic digest (≤1.5k tokens) + ≤10 evidence snippets + current Observations; output = rewritten Observations with per-line evidence citations (mem0-style reconcile, Honcho-style grounding — no personality labels). Unavailable/offline → skip silently, events fold into next session. Done when: a session triggers exactly one call within budget, output validation rejects oversized/markerless responses keeping the prior version, and offline sessions degrade to Facts-only.
- [x] **T4 Inject into Learning prompts** — `profile_excerpt()` (≤700 tokens, Facts summary + Observations only) appended via `system_append` in module generation, tutor, and quiz/flashcard explain paths, alongside the existing `snapshot_prompt()` block; empty until evidence exists. Done when: prompt-assembly tests show the excerpt present and capped for an evidenced subject and absent for a fresh one.
- [x] **T5 Inject into review prompts** — `review-preferences.md` excerpt appended via `system_append` in plan, grill, assess, and chat commands (e.g. observed preference for terse plans or deeper grill follow-ups). Done when: the same assembly tests pass for all four review call sites, and the FakeProvider suite proves no extra model calls were added.
- [x] **T6 Settings surface** — a "Your profile" Settings section rendering the three files: view, edit user notes inline, "reset auto sections" (per file, via ConfirmDialog — never window.confirm), and a master "Adaptive profile" toggle that disables capture, reflection, and injection together. Done when: edit persists, reset clears only auto sections, and toggling off provably stops event writes and excerpt injection.
- [x] **T7 Cost + privacy audit** — assert the per-session budget (≤1 reflection call, ≤4k in/1k out) with a counting FakeProvider over a scripted heavy session; confirm no raw study content or repo source enters profile files (signals and citations only); document the budget here. Done when: the budget test is green and a file-content audit test rejects any profile write containing intake text.
- [x] **Tend Phase verification** — run two real study sessions and one repo-review session, then inspect the MD files: Facts match SQL, Observations cite evidence, a manual user note survives the next reflection, tutor and grill visibly adapt. Done when: all behave, suites green, and PLAN.md is updated.

## Watch for (this phase)
- Reflection is the only model spend here — if it creeps to mid-session or per-interaction calls, the design has failed its constraint; keep it end-of-session-or-batched, behind a single budget-tested entry point.
- The model writes *prose about the user*; keep the existing `profile.rs` discipline — bounded, evidence-cited behavioral facts, never "you are a visual learner" typing, and the user section is sacred (validate before every write, discard on any violation).
- Haiku-quality output can be sloppy with markers/caps — validation must fail closed (keep old Observations), and the whole feature must be a no-op when Claude is unavailable or the toggle is off, so Learning mode never blocks on the profile.
- When a real Ollama provider lands, route reflection to it by default ($0) but gate on the Phase 17 capability check — a weak local model that mangles the reconcile prompt should fall back to skipping, not corrupting the profile.

## Deviations (recorded at close, 2026-06-10)
- T2: shipped the two global files (learner-profile.md, review-preferences.md); per-subject addenda (subjects/<slug>.md) deferred until real evidence shows the global learner file is too coarse — the schema (subject_id on every event) already supports them.
- T3: reflection triggers are app-launch + mode-switch (fire-and-forget, gated at 15+ events) rather than a window-close hook — close hooks are unreliable in webviews and the gate makes opportunistic calls free. A "Reflect now" button in Settings covers the impatient path.
- T7 note: the very first run of the gating test accidentally routed to the real CLI before the stub config was wired (one haiku-tier call); fixed by constructing ModelConfig::Local in the test setup — the suite now provably never spends credits.
- Tend's "two real study sessions" remains for live use: the deterministic capture + injection paths are unit-tested, but the first genuinely adapted tutor session happens when Sam studies.
