# Phase 17 — Settings truth & data integrity
Status: done
Goal: Every setting does what it claims, every save either persists or visibly fails, and the spaced-repetition scheduler actually schedules.
Depends on: Phase 16 (events/cancel infra). Findings cited as A# live in `.planning/AUDIT-2026-06-09.md`.

## Tasks
- [x] **T1 provider_for() everywhere** — grill, plan, learning, cards, and assess route through `provider_for()` instead of hardcoding `ClaudeCodeProvider` (A4/A11: "Local (stub)" currently still spends Claude credits in 5 of 6 features). Done when: a Rust test with the stub provider configured asserts no ClaudeCodeProvider invocation from any feature.
- [x] **T2 Capability-gate the stub** — with Local (stub) selected, generation features show an inline notice ("Local stub returns placeholder output — switch to Claude for real generation") instead of silently producing garbage. Done when: a test renders the notice under stub config and confirms generation is gated. (A real Ollama provider is deliberately out of scope — candidate Phase 20.)
- [x] **T3 History budget keeps the newest turns** — the chat context trimmer drops oldest-first; today it truncates the newest turns, making the model blind to the recent conversation in long chats (A12). Done when: a Rust test with an over-budget history asserts the most recent turns survive.
- [x] **T4 FSRS due queue (Anki-style)** — a `list_due` query (due ≤ now, soonest first) plus new/never-graded cards up to a 20-card session cap; FlashcardsPane consumes the queue; empty queue shows "Nothing due — next card due {date}"; grades update FSRS state only for queued reviews, which also kills the zero-elapsed re-grade corruption (A48: due dates are computed, persisted, indexed — and never read). Done when: tests cover ordering, the cap, the empty state, and that an immediate re-grade of a non-queued card writes nothing.
- [x] **T5 Plan updates dedupe decisions** — plan update/rebuild upserts decisions (matched by stable key) instead of re-inserting every one as a new active row (A6: every rebuild currently duplicates the whole Decisions record). Done when: a Rust test runs update twice and asserts a stable decision count with statuses preserved.
- [x] **T6 Transactional saves; no silent persistence failures** — flashcard/quiz set writes and tutor turns (user message + assistant reply) wrap in transactions so a mid-loop failure leaves no partial/orphaned rows (A27/A31/A35); a failed assistant-reply persist surfaces as an event/notice instead of vanishing on restart (A28). Done when: injected mid-write failures leave zero partial rows and the failure is visible in the UI.
- [x] **T7 Transactional, re-runnable migrations** — wrap the base schema batch and `migrate_v2` in transactions (or add `IF NOT EXISTS` guards) so a crash mid-migration can't permanently brick the DB at next launch (A49). Done when: a Rust test starts from a half-migrated DB and the app opens.
- [x] **T8 SECURITY.md trust statement** — document the deliberate decision: imported repos and uploaded documents are treated as *trusted input*, and model calls keep WebFetch by design; call out that Assess is the surface where third-party repos are the expected input, so users importing untrusted repos should understand the tradeoff (A8 accepted-as-risk). Done when: the section exists and matches actual behavior.
- [x] **Tend Phase verification** — switch to Local stub and touch every feature (no credits spent, notices shown); run a 50-turn chat and confirm recent context sticks; grade a deck and confirm the queue/empty state; rebuild a plan twice. Done when: all pass, suites green.
