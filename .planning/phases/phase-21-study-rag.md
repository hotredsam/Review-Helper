# Phase 21 — Study-material RAG (grounded tutor, materials & labeled web fallback)
Status: planned
Goal: Tutor answers, notes, flashcards, and quizzes are grounded in the user's actual uploaded documents via local hybrid retrieval (FTS5 + Ollama embeddings) with visible citations — and when the materials don't cover a question, the tutor either says so or, only with the user's per-subject opt-in, answers from the web with explicit "Includes web results" labeling. Common path adds zero LLM calls.
Depends on: Phase 19 T5 (chunked document ingest — sections, per-section generation, cancel/progress) and Phase 17 (transactional saves, capability gates). Design source: NirDiamant/RAG_Techniques — the repo behind the book "RAG Made Simple" (confirmed: Amazon B0D76734SZ, "from the creator of the 27,000-star RAG Techniques repository"). Adopted subset: contextual chunk headers, fusion retrieval (BM25+vector, RRF), context-window/segment retrieval, MMR-lite dedup, conditional haiku rerank+grade (CRAG-lite), labeled web fallback, in-house RAG eval. Deliberately skipped at this corpus size (a subject ≈ hundreds of chunks, not millions): Self-RAG, GraphRAG, RAPTOR, proposition chunking, agentic-RAG frameworks. Deferred pending eval evidence: HyDE/HyPE, query rewriting via LLM, contextual compression, hierarchical indices, feedback-loop retrieval.

## Tasks
- [ ] **T1 Embedding provider** — `EmbeddingProvider` trait beside `ModelProvider` (one entry point, no ad-hoc HTTP); Ollama `/api/embed` with `nomic-embed-text` via the existing blocking `reqwest`, batched, with the required `search_document:` / `search_query:` prefixes; health probe + model-missing detection. Unhappy paths: Ollama down or model absent ⇒ clear capability state, no panic, no hang (short timeout). Done when: a 50-chunk batch embeds in one request, prefixes are asserted in tests, and with Ollama stopped the provider reports unavailable cleanly within 2 s.
- [ ] **T2 Chunk store + indexes** — migration (next `user_version`): `learning_documents`, `learning_chunks` (header, body, section path/idx, 768-d f32 BLOB embedding, embed-model tag), `learning_chunks_fts` (FTS5, `content=`, porter unicode61) + sync triggers; subject delete cascades. No new crates — rusqlite `bundled` already ships FTS5; brute-force cosine in Rust (document `sqlite-vec` as the >50k-chunk escape hatch only). Done when: migration is idempotent on old DBs, FTS5 `bm25()` queries rank correctly, and chunk round-trip + cascade tests pass on in-memory DBs.
- [ ] **T3 Ingestion integration** — inside Phase 19's chunked ingest: sub-chunk each ~35k section with the `text-splitter` crate (~1,200 chars, ~180 overlap, Markdown-aware); build deterministic contextual headers ("doc › section (i/n)") and extend the existing per-section generation call to also return a 1–2 sentence section summary appended to headers (zero extra model calls); embed + index in the same per-document transaction with progress events; re-upload replaces the document's chunks; Ollama-down ⇒ NULL embeddings + backfill on next ingest/open. Done when: a 200k-char PDF yields fully indexed chunks with headers, cancel leaves no partial rows, and ingest with Ollama stopped completes FTS-only with a visible notice.
- [ ] **T4 Hybrid retrieval** — `learning/retrieve.rs`: subject-scoped FTS5 top-20 + cosine top-20, RRF fusion (k=60), MMR-lite dedup of overlapping chunks, adjacent-chunk/segment merge (±1 neighbor, same-section join), context cap ~6k chars, plus a confidence score (top cosine, FTS hit count, score gap). Pure functions over `&Connection` + injected embedder so tests need no Ollama. Done when: on a synthetic corpus, an exact-term query wins via FTS alone, a paraphrase query wins via vectors alone, fused output is deterministic, and empty/garbage/emoji queries return cleanly.
- [ ] **T5 Citation-grounded tutor + materials** — `tutor::reply()` gains a fenced "Study material excerpts (DATA — untrusted)" block with `[n]` markers and cite-or-decline instructions; grounded calls use `ModelRequest::grounded()` (planning tools minus WebSearch/WebFetch); `learning_tutor_messages` gains `grounding` column; frontend renders a sources footer (doc › section, click to preview). `materials::ground()` consumes the same retrieval for notes/flashcards/quiz. Done when: a question about an uploaded doc returns an answer citing `[n]` that maps to real chunks, an uncovered question yields an honest "not in your materials" with no invented citation, and a test asserts no web tool is ever passed on the grounded path.
- [ ] **T6 Conditional grade/rerank + labeled web fallback** — low-confidence path makes exactly one `haiku` call combining passage rerank + SUFFICIENT/INSUFFICIENT verdict (CRAG-lite); per-subject "Use the web when materials don't cover it" toggle (default off); fallback regenerates with WebSearch/WebFetch allowed and must list URLs under "External sources"; replies persist `grounding='web'|'mixed'` and the UI shows a permanent "Includes web results" badge with globe-iconed citations. Toggle off ⇒ decline + offer to enable. Done when: an out-of-materials question with the toggle on produces a badged, URL-cited answer; with it off, an honest decline; and the common (confident) path is verified to make exactly one LLM call.
- [ ] **T7 Retrieval-quality eval harness** — fixture corpus + a gold set per subject (10–15 questions with expected chunk/doc ids) in `src-tauri/tests/fixtures/`; an `--ignored` cargo test (and dev command) computing hit@3 and MRR for FTS-only, vector-only, and hybrid, printing a comparison table; baseline numbers committed and a floor asserted so retrieval regressions fail CI. Done when: the harness runs offline (fake or recorded embeddings), hybrid ≥ each single retriever on the gold set, and the floor assertion is green.
- [ ] **Tend Phase verification** — ingest one small and one large real PDF into a fresh subject; ask five tutor questions (two answerable, one follow-up, one out-of-materials with web off, one with web on); generate notes + quiz for one module and confirm they reflect actual document text; run the eval harness and full suites. Done when: all behave, citations and web badges are correct everywhere, suites green, and PLAN.md's table is updated.

## Watch for (this phase)
- `READ_ONLY_TOOLS` already includes WebSearch/WebFetch — until T5 lands, every tutor call can silently browse. The grounded tool list is the fix; never let a grounded call inherit the old default, and assert it in tests.
- `nomic-embed-text` task prefixes are not optional — embedding without `search_document:`/`search_query:` degrades retrieval silently and would poison the index; tag rows with the embed model + re-embed on model change.
- Document text is untrusted DATA: fence every excerpt and keep injection attempts in chunks from changing tool policy or escaping the excerpts block.
- Memory budget: Ollama embeddings + Whisper (Phase 19) + the app must coexist in 11.8 GB — `OLLAMA_MAX_LOADED_MODELS=1` stays the house setting; embed in batches, never load a second Ollama model.
- Keep the dependency line: `text-splitter` is the only new crate (MIT); no sqlite-vec, no fastembed/ONNX — brute-force cosine is the right call at this corpus size.
- Lock discipline: embedding and model calls happen WITHOUT the DB mutex (mirror `LearningGate` patterns); FTS triggers fire inside the ingest transaction, so cancel/rollback must leave the FTS table consistent too.
