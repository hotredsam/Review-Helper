# Screenshots — capture guide

The project [`README.md`](../../README.md) embeds the six images below. Drop them in
**this folder** with these exact filenames and they render automatically — no README edit needed.

| Filename | Screen | What to show |
|---|---|---|
| `overview.png` | **Overview / State pane** | The assessment scorecard — six vibecoding dimensions (radar), the production-readiness gauges, the overall score, and the hygiene list. The most impressive single screen; capture it on a project that has been analyzed + assessed. |
| `grill.png` | **Grill pane** | A few repo-specific questions (each with its draft "recommended" answer), the depth slider, and the **Detail Coverage** meter showing progress. |
| `plan.png` | **Plan pane** | The phased plan with a couple of phases expanded (tasks + "Done when"), the Decisions and Stack sections, and the GitHub **Sync** panel. |
| `chat.png` | **Chat pane** | A short conversation with the **Pending suggestions** panel visible (the Approve / Dismiss chips). |
| `understand.png` | **Understand hub** | The concept-card grid across domains, ideally with one card's What / When / Why expanded. |
| `themes.png` | **Themes** | One screen shown across the four themes (`light` / `dark` / `midnight` / `sand`) — a 2×2 montage, or just the theme switcher open. |

## How to capture (macOS)

- **A single window, cleanly:** press `⌘⇧4`, then `Space`, then click the Review Helper window. macOS captures just that window (with a drop shadow) to your Desktop. Rename it to the filename above and move it here.
- **A precise region:** `⌘⇧4` then drag. Use `⌘⇧5` for more control (and to pick where it saves).
- The app window is **1200×800**; capturing on a Retina display yields crisp 2× images, which look best on GitHub.
- For the data-rich panes (`overview`, `grill`, `plan`), capture a project that already has a generated plan + assessment so the charts and lists are populated rather than empty states.

## Notes

- Keep them reasonably sized (a few hundred KB each is plenty after macOS PNG compression).
- PNG is preferred; the README `<img>` tags reference `.png`.
- If you'd rather I embed/lay them out differently once they exist, just say so.
